## Editorial Guidelines

### Title Formula
Use one of:
- `How to [Task] in [Framework] — [Outcome]`
- `[Comparison]: [Option A] vs [Option B] in [Year]`
- `Building [X] with [Library] — [Outcome]`
- `[Concept]: [Opinionated Take]`

### SEO Checklist Per Post
- [ ] Primary keyword in H1 and first 100 words
- [ ] Secondary keyword in one H2
- [ ] 2–4 internal links to docs/guides
- [ ] 1–2 external links to authoritative sources
- [ ] Code sandbox or live demo embedded where possible
- [ ] Open Graph image (1200×630) generated per post
- [ ] Published with `datePublished` schema.org markup
- [ ] Cross-posted to Dev.to within 3 days (canonical back to gettraek.com)

### Author Guidelines
- Use first-person plural ("we built") for company posts
- Use first-person singular for personal technical deep-dives
- Always include a working code example (CodeSandbox or StackBlitz)
- Keep intro under 100 words — get to the code fast
